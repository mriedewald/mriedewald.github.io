How to use it:

Query Input tab

CAREFUL: correlation search can be expensive as there are n2 possible pairs for n summaries. E.g., if there are 100,000 summaries, Scolopax might have to explore up to 10 billion pairs!

  1. Select species of interest
    1. Use control+click to add another species
    2. Use shift+click to add range of species
  2. Select region and partitioning of interest
  3. Compose a join conditions to find related pairs of summaries
    1. Correlation finds similar summaries, anti-correlations finds those most different from each other.
    2. The species join condition limits the summary pairs considered to those where both species are (not) the same.
    3. The region join condition limits the summary pairs considered to those that are (not) for the same geographical region.
    4. The epsilon value eliminates poor matches. If you are not sure how to set it, choose a large value, e.g., 100, for correlation search and zero for anti-correlation search. Then decrease (correlation search) or increase (anti-correlation search) it to reduce result size.
  4. Submit
  5. The Predefined Queries section allows storing and re-using of queries for future use. Re-execution will be faster, because previous results are stored in HBase (but might be deleted at some point when the database runs out of space.)

Results tab

  1. View the summary pairs patterns found, sorted by their (anti-)correlation, measured as the distance between the lines after shifting by mean (for shift-invariance).
    1. When hovering over the plot line or title, the corresponding region is highlighted on the map.
  2. Eliminate results from the list if desired
    1. In the filter panel, choose which results to remove. E.g., selecting a species will remove all results for the species.
    2. Filters can be added and removed as desired.
  3. The settings panel allows plot settings to be changed. To export a plot, un-select "show interactive plots".
  4. The plots at the bottom of the results panel visualize the distribution of input and output over the Reduce tasks (which perform the actual join work) during join computation.

At any time, it is possible to go to another tab by simply clicking on it. However, if the browser's forward or back buttons are used, the previous state is lost.
